## What lives here

These modules cover the rest of the public package surface: runtime helpers, transport utilities, caching, crypto, eventing, and polling.

## Environment and storage

| Entrypoint | Main export | Purpose |
| --- | --- | --- |
| `bytekit/env-manager` | `EnvManager` | Access environment values across runtimes |
| `bytekit/storage-utils` | storage helpers | Browser and storage-related helpers |

### `EnvManager`

Main methods:

- `get(name)`
- `require(name)`
- `isProd()`

Example:

```ts
import { EnvManager } from "bytekit";

const env = new EnvManager();

const apiUrl = env.require("API_URL");
const locale = env.get("APP_LOCALE") || "en";
```

| Entrypoint | Main export | Purpose |

| Entrypoint | Main export | Purpose |
| --- | --- | --- |
| `bytekit/cache-manager` | `CacheManager` | General cache strategies |
| `bytekit/compression-utils` | `CompressionUtils` | Compress and decompress data |

| `bytekit/diff-utils` | `DiffUtils` | Diff objects and arrays |

## `StreamingHelper`

Main methods:

- `streamJsonLines(endpoint, options?)`
- `streamSSE(endpoint, options?)`
- `downloadStream(endpoint, options?)`

Example:

```ts
import { StreamingHelper } from "bytekit";

const result = await StreamingHelper.streamJsonLines("/api/logs", {
  onChunk: (chunk) => console.log(chunk),
});
```

## `WebSocketHelper`

Main methods:

- `connect()`
- `send(type, data)`
- `on(type, handler)`
- `onError(handler)`
- `request(type, data, responseType?)`
- `close()`
- `isConnected()`
- `getState()`

Example:

```ts
import { WebSocketHelper } from "bytekit/websocket";

const socket = new WebSocketHelper("wss://example.com/socket");
await socket.connect();

socket.on("price:update", (payload) => {
  console.log(payload);
});
```

## `CacheManager`

Use it when you want broader caching strategies than request-specific caching.

Typical use cases:

- Cache computed values
- Layer memory and persistent storage
- Track hit rate and cache size

Example:

```ts
import { CacheManager } from "bytekit/cache-manager";

const cache = new CacheManager({
  defaultTTL: 60_000,
  maxMemorySize: 100,
  enableLocalStorage: true,
});

const stats = await cache.getOrCompute("dashboard-stats", loadStats);
```

## `CryptoUtils`

Use it for:

- Hashing
- HMAC generation
- UUID generation
- Secure token workflows
- Constant-time comparison

Example:

```ts
import { CryptoUtils } from "bytekit/crypto-utils";

const token = CryptoUtils.generateToken(32);
const uuid = CryptoUtils.generateUUID();
const hash = await CryptoUtils.hash("payload");
```

## `EventEmitter`

Main methods:

- `on(event, listener)`
- `once(event, listener)`
- `off(event, listener)`
- `emit(event, data)`
- `emitSync(event, data)`
- `onError(listener)`
- `listenerCount(event)`

Example:

```ts
import { EventEmitter } from "bytekit/event-emitter";

type Events = {
  "job:done": { id: string };
};

const emitter = new EventEmitter<Events>();
emitter.on("job:done", ({ id }) => console.log(id));
await emitter.emit("job:done", { id: "42" });
```

## `PollingHelper`

Use it when a workflow needs repeat checks with consistent timing and control over stop conditions.

Main methods:

- `start()`
- `startWithAbort()`
- `abort()`

Example:

```ts
import { PollingHelper } from "bytekit/polling-helper";

const poller = new PollingHelper(
  () => fetchJobStatus(),
  {
    interval: 5000,
    maxAttempts: 20,
    stopCondition: (result) => result.status === "done",
  }
);

const result = await poller.start();
```

## `CompressionUtils`, `DiffUtils`, and file upload helpers

These modules are useful when you need:

- payload compression or decompression
- deep comparisons between objects or arrays
- upload flows with helper abstractions

Prefer their narrow entrypoints when you only need one concern.

<Note>
  These helpers are intentionally modular. Most apps only need a small subset, so prefer the narrow entrypoint for each use case.
</Note>
