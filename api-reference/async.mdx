## Module entrypoint

- `bytekit/async`

## What this module is for

Use `bytekit/async` when you need control over timing, retries, and concurrency outside the HTTP client itself.

## Timing

| Export | Purpose |
| --- | --- |
| `sleep(ms, signal?)` | Delay execution with optional abort support |
| `timeout(promise, ms, message?)` | Add a timeout to a promise |
| `withTimeout(fn, ms, message?)` | Wrap a function with timeout behavior |

### Example: timeout around a promise

```ts
import { timeout } from "bytekit/async";

const report = await timeout(fetchReport(), 5000);
```

## Retry and execution control

| Export | Purpose |
| --- | --- |
| `retry(fn, options?)` | Retry a failed async operation |
| `debounceAsync(fn, waitOrOptions)` | Debounce an async function |
| `throttleAsync(fn, waitOrOptions)` | Throttle an async function |

### Example: retry a flaky operation

```ts
import { retry } from "bytekit/async";

const payload = await retry(() => fetchData(), {
  maxAttempts: 5,
  baseDelay: 1000,
  backoff: "exponential",
});
```

### Example: debounce async search

```ts
import { debounceAsync } from "bytekit/async";

const debouncedSearch = debounceAsync(async (query: string) => {
  const response = await fetch(`/api/search?q=${query}`);
  return response.json();
}, 300);
```

## Concurrency and coordination

| Export | Purpose |
| --- | --- |
| `parallel(tasks, options?)` | Run tasks with optional concurrency limit |
| `sequential(tasks, options?)` | Run tasks one after another |
| `race(promises)` | Resolve with the first successful or settled candidate according to module semantics |
| `allSettled(promises)` | Collect fulfilled and rejected results |

### Example: bounded concurrency

```ts
import { parallel } from "bytekit/async";

const results = await parallel(
  files.map((file) => () => uploadFile(file)),
  { concurrency: 5 }
);
```

### Example: multi-source fallback

```ts
import { race, sleep } from "bytekit/async";

const result = await race([
  fetchFromPrimaryRegion(),
  fetchFromSecondaryRegion(),
  sleep(5000).then(() => {
    throw new Error("Timeout");
  }),
]);
```

## Error types

The async module also exposes specialized error classes:

- `TimeoutError`
- `AbortError`
- `RetryError`

## Typical options

### `retry`

| Option | Purpose |
| --- | --- |
| `maxAttempts` | Maximum tries |
| `baseDelay` | Starting delay |
| `maxDelay` | Delay cap |
| `backoff` | `linear`, `exponential`, or custom function |
| `shouldRetry` | Predicate to keep retrying |
| `signal` | Abort signal |

### `parallel`

| Option | Purpose |
| --- | --- |
| `concurrency` | Number of tasks allowed at once |

### `sequential`

| Option | Purpose |
| --- | --- |
| `continueOnError` | Keep executing after failures |

## Combined example

```ts
import { parallel, retry, timeout } from "bytekit/async";

const result = await retry(() => timeout(fetchData(), 5000), {
  maxAttempts: 4,
  baseDelay: 250,
});

const outputs = await parallel(jobs, {
  concurrency: 3,
});
```
